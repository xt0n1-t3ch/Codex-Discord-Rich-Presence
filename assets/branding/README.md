# Branding Assets

This folder contains project visuals for Codex Discord Rich Presence (`codex-discord-presence`).

- `project-mark.svg`: repository/project icon.
- `social-card.svg`: true-black hero banner with one-line centered title.
- `surface-map.svg`: true-black runtime map with deterministic routing overview.
- `credits-ribbon.svg`: minimal true-black credits banner with embedded app icon for reliable Markdown preview rendering.
- `codex-app.png`: Codex App icon source used for Discord assets and executable branding.
- `chatgpt-app.jpg`: ChatGPT-style black desktop identity used by the canonical desktop design toggle and downstream Pulse preview. SHA-256: `F7B325F08B0C2D11E19AE18F4A677B9EC80A973E2D8DF968FA8E5124B90CF71B`.
- `codex-readme-hero.png`: README hero image in the soft Codex App visual language.
- `codex-feature-strip.png`: README feature strip for Discord, privacy, Rust runtime, cache, context, and dashboard concepts.
- `../readme/badges/*.png`: cropped README badge capsules for release, CI, OpenAI Codex, Discord RPC, Rust, Ratatui, local-first, and platform support.
- `../readme/badges/release-v1.7.3.png`: versioned release capsule; README and preflight version checks advance together.
- `../readme/icons/*.png`: cropped README heading icons styled as soft Codex gradient tiles.

## Badge Policy

The README uses local cropped PNG badge capsules instead of external badge generators. This keeps spacing, typography, glow, rounded corners, and the Codex visual style consistent in GitHub dark and light previews.

Brand cues are preserved inside the badge art:

- OpenAI: black/white mark treatment.
- Discord: blurple `#5865F2`.
- Rust: black mark treatment.
- Windows: Microsoft blue `#0078D4`.
- macOS: black Apple treatment.
- Linux: Tux yellow `#FCC624`.

External text links remain below the badge row for clickable source attribution.
